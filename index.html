Arquitectura de Software MVC

Modelo - Vista - Controlador

MVC es un patrón de arquitectura de software que organiza una aplicación en tres componentes principales, optimizando el desarrollo, la mantenibilidad y la escalabilidad.

Modelo MySQL
Se encarga de los datos y la lógica de negocio. Gestiona la información de forma directa, realiza consultas en bases de datos y aplica rigurosamente las reglas operativas del sistema.
Vista QT
Es la interfaz con la que interactúa el usuario de forma visual. Se encarga de formatear y presentar los datos que recibe del modelo, libre de lógica de negocio o procesos internos.
Controlador Python
Actúa como el orquestador principal e intermediario entre el modelo y la vista. Recibe las interacciones del usuario, procesa las solicitudes correspondientes y dictamina qué datos se deben actualizar o mostrar.
